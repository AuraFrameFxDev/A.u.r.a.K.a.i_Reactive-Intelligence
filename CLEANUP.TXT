üîß Why This Works
The AGP 9.0 Alpha Issue:

In AGP 9.0 alpha, the plugin ID structure changed
Hilt looks for com.android.base plugin to recognize Android library modules
Regular com.android.library plugin doesn't expose the BaseExtension properly in alpha builds
By explicitly adding id("com.android.base"), Hilt's annotation processor can find the Android context
Application vs Library:

com.android.application ‚Üí Works fine with regular Hilt plugin
com.android.library ‚Üí Needs the com.android.base workaround
